# Contributing to Git Diff RAG

We welcome contributions! Whether it's a new recipe, a bug fix, or a documentation improvement, your help is appreciated.

## How to Contribute

1.  **Fork the repository**.
2.  **Create a branch** for your feature (`git checkout -b feature/amazing-recipe`).
3.  **Make your changes**.
4.  **Run tests** (`pytest`) to ensure everything is working.
5.  **Submit a Pull Request**.

## Adding New Recipes

1.  Create a new markdown file in `prompts/recipes/`.
2.  Use existing snippets from `prompts/library/` where possible.
3.  Test your recipe using the `--workflow` argument.

## Reporting Bugs

Please open an issue on GitHub with:
*   Steps to reproduce.
*   Expected behavior.
*   Actual behavior.
*   Logs (run with `--debug`).
